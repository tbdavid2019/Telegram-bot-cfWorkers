# 長期記憶儲存格式設計

## 儲存策略

### KV 模式（預設）
- `memory:global` - 全域知識庫
- `memory:user:{telegramId}` - 個人記憶（例：`memory:user:650289664`）

### R2 模式
- `memory/global.md` - 全域知識庫
- `memory/user_{telegramId}.md` - 個人記憶（例：`memory/user_650289664.md`）

---

## 記憶更新策略

### 自動提取規則
1. **身份識別**: 「我是...」、「我叫...」
2. **偏好表達**: 「我喜歡...」、「我習慣...」
3. **重要事實**: 特定日期、地點、人名
4. **習慣行為**: 常用指令、查詢模式

### 更新時機
- 每次對話結束後（延遲 1 秒執行，避免阻塞回應）
- 使用 LLM 分析對話，提取值得記憶的資訊
- 自動 merge 到現有記憶，不覆蓋

---

## 實作細節

### 讀取流程
```
1. 檢查 ENABLE_LONG_TERM_MEMORY
2. 根據 MEMORY_STORAGE_MODE 選擇 KV/R2
3. 並行讀取 global + user_{id}
4. 組合成系統提示詞
```

### 寫入流程
```
1. LLM 對話完成
2. 使用 LLM 分析對話內容
3. 提取新資訊
4. Merge 到現有記憶
5. 寫入 KV/R2
```

### 錯誤處理
- KV/R2 讀取失敗 → 返回空記憶，繼續運作
- 記憶格式錯誤 → 記錄 log，使用預設格式
- 寫入失敗 → 記錄 log，不影響對話
